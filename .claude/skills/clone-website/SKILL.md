# Clone Website

$ARGUMENTS を、営業用リメイクテンプレの参考サイトとして取り込む。

この repo では、弱い営業先サイトをそのまま複製するためにこの skill を使わない。強い参考サイトを取り込み、あとで営業先ごとに差し替えられるテンプレ初稿にするために使う。

## 事前条件

1. ブラウザ自動操作が必要
2. URL を開始前に検証する
3. スクリーンショットと調査メモは `docs/research/<slug>/` に保存する
4. 取り込んだテンプレとアセットは `assets/reference-templates/<slug>/` に保存する
5. 必要なフォルダがなければ作成する

## 既定の対象範囲

- 対象: レイアウト、セクション順、余白、配色、タイポグラフィ、アイコン、導線、インタラクション、レスポンシブ挙動
- 対象外: バックエンド、認証、実フォーム送信、分析、公開用 SEO
- 取り込み時の前提: 参考サイトのブランドは適宜外し、再利用可能なテンプレ初稿に寄せる

## 手順

1. 参考サイトを PC とスマホで確認する
2. 全体スクリーンショットとセクションごとのスクリーンショットを取る
3. デザイントークン、アセット、セクション順、主要インタラクションを抜き出す
4. 出力先にテンプレ初稿を作る
5. 作りながら、明らかに再利用できるセクションや UI を分ける
6. 最後に短いメモを残す

メモに含めるもの:

- 参考サイトから取り込んだ構造
- クライアント利用前に差し替えが必要な項目
- 業種横断で再利用できそうなセクション

## ルール

- ユーザーが厳密クローンを明示しない限り、参考サイトのロゴ、社名、住所、電話、証言、主張をテンプレに残さない
- 1ブランドへの忠実さより、再利用しやすい構造を優先する
- 後で部品化しやすいよう、セクション境界を明確にする
- スクリーンショットと調査メモは監査できる形で残す
- 複雑なインタラクションがある場合は、再現前に挙動を記録する

## 完了時の報告

- テンプレ出力先
- 調査メモの保存先
- 再利用可能と判断したセクション
- まだ差し替えが必要な項目
- build または preview の状態
